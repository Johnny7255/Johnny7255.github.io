预备知识————树的基本定义

  （递归定义方式） 树(tree)是一些节点的集合，这个集合可以为空集，也可以不是。若不为空集，则树是由一个根节点，以及 0 或多个非空子树组合而成的。这些子树通过一个有向边(edge)同一个根节点 r，而这些子树的根节点为 r 的子节点(child)，而 r 是每棵子树根节点的父节点(parent)，而子树为 0 个的节点，被称为叶子节点(leaf)。
  其是N 个节点与N-1 条有向边的集合。有相同父节点的边被称为兄弟节点(siblings)。从节点 Na 到节点 Nb 的序列被称为路径(path),路径的**长(length)是这条路径上经过的边数。
  深度(depth),是从根节点到某个节点的路径长，一个节点的高度(height)**则是从该节点到能到到的叶节点的最长的路径长。如下图 B 和 C 的深度都为 1，但是 B 高度为 2，C 为 1

二叉树简介

  对于大量的数据增删改查，链表的线性访问太慢，而哈希表的内存占用太大并且难以编写合理的哈希函数避免冲突。而二叉树是一种比较简单的数据结构，并且有较好的访问能力，理想条件下对于增删改查等操作的时间复杂度都为
O(log N)。
  而二叉树是一种每一个节点都不能有多余两个子节点（可以只有一个子节点）的一种特殊结构，最顶端的节点称为根节点（没有父节点），而末尾称为叶节点（没有子节点）。

二叉树性质

  1. 一棵平均二叉树的深度要比节点个数 N 小得多，一棵二叉树平均深度为 O(N^(1/2)),但是也存在 O(N)的情况(为链表)
  2. 二叉树第 i 层上的结点数目最多为 2*(i-1)(i>1)
  3. 深度为 k 的二叉树至多有 2^(k) - 1 个结点（k>=1）,叶子节点为 2^(k - 1) (满二叉树情况)
  4. 包含 n 个结点的二叉树的高度至少为( log2(n) )+1
  5. 在任意一棵二叉树中，若叶节点的个数为 n0，度为 2 的结点数为 n2，则 n0=n2+1
  6. 子节点个数为 n 的完全二叉树有最少的总节点数

常见二叉树

完全二叉树与完美二叉树

  这两种二叉树是最基本的两种二叉树，它们是更好理解接下来二叉树的基础前提。
  满二叉树：除了最后一层的叶节点没有子节点外，其他节点都有两个子节点。
  完全二叉树：一棵深度为 k 的有 n 个结点的二叉树，对树中的结点按从上至下、从左到右的顺序进行编号，如果编号为 i（1≤i≤n）的结点与满二叉树中编号为 i 的结点在二叉树中的位置相同，则这棵二叉树称为完全二叉树。

二叉堆(Binary Heap)

  二叉堆是一种常见的数据结构，其普遍作为优先队列(priority queue)使用 优先队列一般不是队列结构 。堆是一棵有堆序性(heap-order property)的完全二叉树（就是子节点排列存在一定顺序）。其支持插入(insert)与出队(dequeue)。（教材这里可能写插入与删除最小操作，我之所以这样写是因为我们通常是利用堆获取某个特殊的节点，这个节点未必是最小，而且删除并不能表达”获取这个节点“的意思，因此我这里选择出队。学习的同学请按照老师教的）。其特点是最满足某个条件的节点永远在首位。其操作最坏时间永远是O(log N)

(二叉堆)

二叉查找树(Binary Search Tree)

  二叉查找树是一种很常见的二叉树，简称BST，其也被称为二叉排序树。其实这应该算是一个大类，AVL 树和红黑树也属于二叉查找树的范畴，所以就简单说一下二叉查找树的基本特点。二叉查找树的左节点的值永远小于父节点，而右节点相反。因为这种有序性，这种书平均深度为 O(log N),增删改查的理想时间复杂度也是 O(log N)。但是对于普通的二叉查找树来说，是存在整个二叉树会变成链表的情况的。为了避免这种情况的发生，于是有了AVL 树与红黑树这两种高级的二叉查找树。

AVL树

  普通的二叉查找树有个缺点，其并没有能确保平衡结构的手段，随着删除和增加的次数增加，树的平衡会逐渐被破坏，子树之间的高度差越来越大，最糟糕情况就变成了链表，增删改查的时间复杂度变为了 O(N)而非我们期待的 O(log N)。而我们为了保持 O(log N)的时间复杂度，保持子树之间的高度差尽可能小是必须的，因此我们需要使用有平衡结构的二叉树。其中AVL 树与红黑树为最出名的自平衡二叉查找树。
  AVL 树是第一个被发明的自平衡树，其要求任意一个节点的两个子树之间的高度相差不能大于 1，而这样也保证了树的深度是 O(log N)。而为了保证这种自平衡性，AVL 树在普通的 BST 的增删操作基础上增加了旋转操作，其操作比普通的 BST 复杂，但是当数据量过大时，其效率会比普通的 BST 高得多，这点牺牲是值得的。

红黑树

  红黑树是一种更加复杂的自平衡二叉查找树，其是 AVL 树的一种变种，对它的操作最坏时间复杂度为 O(log N)，是非常理想的一种二叉查找树。其每个结点被标记未红和黑这两种颜色之一。

其着色要求如下：

  1. 每一个节点不是红色的就是黑色的。
  2. 根节点为黑色。
  3. 如果一个节点是红色的，那么其子节点必须是黑色。
  4. 从一个节点到一个 null 叶节点的路径必须包含相同数目的黑色节点。

  而这个着色发着，让红黑树的高度最多为 2log(N+1)

树的遍历

  因为树不是普通的链表结构，更不是数组结构，因此我们没办法直接一个节点接着一个节点的遍历树。而树常见的遍历方式为先序遍历(preorder traversal)，中序遍历(inorder traversal)，后序遍历(postorder traversal)和层序遍历(level traversal)。其中前三种遍历方式采取的是DFS策略，而层序遍历(level traversal)采用的是BFS策略。
