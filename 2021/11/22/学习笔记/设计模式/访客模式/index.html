访客模式(笔记)

2021年11月22日 551 字 大概 2 分钟

访客模式

GOF 定义

表示一个作用于某对象结构中的各元素的操作，使你可以在不改变各元素的类的前提下定义作用于这些元素的新操作。

概念

这个模式允许在不改变一个类结构的情况下给某些类加功能，在功能需要经常变化的时候非常有用。理想情况下，访客定义特定类别的方法，这些方法可以让这个类有新的功能。其是将算法与对象结构分开，然后可以在新的层次结构里面加入新的功能，其完美符合 Open/Close 原则。

情景描述

当你打车的时候，你告诉了司机你的目的地，那么车的控制权和路线的选择权就全部交给司机了，司机能走你平常走的路，可能走你不熟悉的路，可能带你跑圈圈。

其很适合支持插件的 API

与组合模式使用

这个模式非常适合同组合模式使用，这种方式特点如下：

  1. 访客可以根据不同的对象可以有不同的访客方法。
  2. 对象可以根据访问者的不同而有不同的新的拓展用法。

Q&A

  1. 什么时候可以考虑使用访客模式？
    当我们需要向一个类新增加功能拓展，而不去改变那个类的结构时，我们可以考虑这个模式。如果存在一些经常需要改变的操作的时候，其也是一个非常合适的解决方案。
  2. 访客模式有什么缺点？
    • 这个模式有点破坏封装性。
    • 如果经常在现有架构上不停添加新的实现类，访问器的层次会变得越来越复杂从而难以维护。
  3. 为什么说访客模式破坏封装？
    因为访客模式可能需要访问各种各样的类中的元素来实现各种各样的功能，但是我们又不能直接强行修改原本的类，所以我们不得不将一些属性暴露给访问器。
