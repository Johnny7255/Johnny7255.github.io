cin 输入流

  cin 表示输入流，可以获得从键盘输入的多个值。**>>被称为提取运算符**，>>可以连续写多个，每个提取运算符后接入一个获得输入值的对象或者对象，其能自动将输入值转为相应数据类型。

cout 输出流

  cout 表示输出流，可以输出整数，实数，字符与字符串。**<<被称为插入运算符**，可以连续写多个，输出的值、转义序列与字符串。

CPP 函数

函数定义

函数由函数头与函数体组成，函数基本定义语法为
<返回值类型> 函数名(参数列表)
{函数语句}

函数调用

调用函数要求先声明再调用，否则在调用函数之前需要在主调函数中中声明函数原型。
声明形式（类似与 java 与 C#中的接口）：
<返回值类型> 函数名(形参表); (形参名可省略)

注意，形参在声明时不会占用内存空间，只有当函数被调用时候才会。

注意事项

  1. 在函数又有原型声明又有定义的时候，默认参数只能在原型声明中使用，而不能在函数定义中指定。
  1. 默认值的定义顺序是从右到左的，不允许中间跳开给值。
    因为函数调用的时候，系统是从左到右顺序进行形实结合的，当参数数目不足时候，系统才会使用默认值进行补齐。

CPP 引用

引用是给程序某个变量一个别名，我们可以通过这个别名操作或者创建某个变量。（可以理解为类型相同，内存共用）
注意，引用不是值，在刚开始声明的时候其并不会被占用内存。
引用在声明时必须初始化。
对引用的修改会改到引用指向的目标。

引用的声明

<标识符>& 引用名 或者 <标识符> &引用名

引用的传递

引用作为参数传递的好处在于，传入一个对象后不会因为按值传递而复制一个对象，从而节约时间与内存。

  const 关键字修饰的被称为常量，即不能被修改的部分。其可以用来修饰方法，成员变量，自定义对象，返回值与函数参数。

const 修饰指针变量

const 修饰指针指向的内容，则这个内容不可为变量。

const 修饰指针，则指针的值不可变，但是指针指向的值的内容可以改变。

const 修饰指针和指针指向的内容，则指针和指针指向的内容都为不可变量。

修饰传递参数与函数返回值

值传递的 const 修饰传递，可以防止传入的参数被修改。

当 const 参数为指针时，可以防止指针被意外篡改。

CPP 中自定义类型的参数传递，需要临时对象复制参数，对于临时对象的构造，需要调用构造函数，比较浪费时间，因此我们采取 const 外加引用传递的方法。而以这种方式被传递的对象，我们不能修改它里面的成员属性。（CPP 传递对象默认按值传递，会复制对象的参数并且重新调用构造函数，而在 Java 与 C#里面，对象是引用类型，因此不存在这种麻烦）

const 修饰类成员函数

const 修饰类成员函数，其目的是防止成员函数修改被调用对象的值，如果我们不想修改一个调用对象的值，所有的成员函数都应当声明为 const 成员函数。

内联函数

内联函数与一般函数不同，内联函数不是调用的时候发生转移的。而是在编译时将函数体嵌入每一个调用语句中的简单函数。相当于不是普通的调用，而是把那一整串代码塞到你调用的地方。而内联的 inline 代码实际上是对编译器的建议，是否内联由编译器决定，若一个函数没有成功内联或者不适宜得内联则会造成不适宜的开销。
优势：节省参数传递，系统栈保护与恢复等开销。
更加具体的讲解可以看这一篇 内联函数的使用及注意点

定义形式

<返回值类型> 函数名(参数列表)
{…}

注意事项

  • 内联函数不适合循环语句与 switch 语句，因为可能会使得函数变得复杂，而导致编译器选择不内联。
  • 内联函数中不能有递归操作(自己想一下如果成功递归内联你的代码会多膨胀)。
  • 内联函数定义必须在第一次调用之前。
  • 函数应当简单，并且使用频率较高。
  • 编译器会决定是否内联，若不能成功内联会造成额外编译开销。
